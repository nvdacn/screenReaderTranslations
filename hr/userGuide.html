1. Uvod

Pružajući povratne informacije pomoću sintetičkog govora i brajice, NVDA omogućuje slijepim osobama i osobama s oštećenjem vida pristup računalima sa windows operativnim sustavom za cijenu ne veću od one koju plaćaju korisnici normalnog vida. NonVisual Desktop Access (NVDA) je besplatan čitač zaslona otvorenog koda za operativni sustav Microsoft Windows. NVDA je razvijen od strane organizacije NV Access, uz doprinose zajednice.

1.1. Glavne Značajke

Nvda omogućuje slijepim i slabovidnim osobama pristup i rad u windows operativnom sustavu te u mnogo aplikacija napravljenih od strane treće strane.

Glavne značajke uključuju:

  • podrška za poznate programe uključujući web preglednike, klijente elektroničke pošte, programe za čavrljanje putem interneta i uredske pakete
  • ugrađen sintetizator govora koji podržava preko 43 jezika
  • izvještavanje o oblikovanju teksta gdje je to podržano kao što su to ime fonta i informacije o bojama, stil i pravopisne pogreške
  • automatsko obavještavanje o tekstu ispod pokazivača miša i neobavezna zvučna indikacija o položaju miša
  • podrška za mnoge brajične retke, uključujući unos kompjutorske brajice za brajične retke koji posjeduju brajičnu tipkovnicu
  • mogućnost pokretanja sa USB memorijskih stickova u cjelosti, ili drugih prijenosnih medija bez potrebe instalacije
  • govorni instalacijski program koji je jednostavan za korištenje
  • preveden na 47 jezika
  • podrška za moderne Windows operativne sustave u 32 i 64 bitnim varijantama
  • mogućnost pokretanja na Windows ekranu za prijavu i ostalim sigurnosnim ekranima
  • podrška za uobičajena sučelja za pristupačnost kao što su Microsoft Active Accessibility, Java Access Bridge, IAccessible2 i UI Automation (UI Automation je podržan samo u windows 7 i novijim inačicama)
  • podrška za windows naredbeni redak i druge aplikacije koje rade u konzoli

1.2. Internacionalizacija

Vrlo je važno da ljudi gdjegod u svijetu, bez obzira na jezik kojim govore, imaju jednak pristup tehnologiji. Osim engleskog, nvda je preveden na 47 jezika uključujući: Afrikanerski, albanski, Amharski, arapski, aragonski, brazilski portugalski, bugarski, katalonski, kolumbijski španjolski, hrvatski, češki, danski, nizozemski, perzijski, finski, francuski, galicijski, grčki, gruzijski, njemački, hebrejski, Hindi, mađarski, islandski, irski, talijanski, japanski, korejski, nepalski, norveški, poljski, portugalski, pendžabski, rumunjski, ruski, srpski, slovački, slovenski, španjolski, švedski, tamilski, tajski, tradicionalni i pojednostavljeni kineski, turski, ukrajinskii vijetnamski.

1.3. Podrška Za Sintetizatore Govora

Osim podrške za mnoge jezike, NVDA može također omogućiti korisniku čitanje sadržaja na bilo kojem jeziku, sve dok korisnik ima sintetizator govora za taj određeni jezik.

NVDA dolazi sa eSpeak-om, besplatnim, otvorenog koda, višejezičnim sintetizatorom govora.

Informacije o drugim sintetizatorima govora koje NVDA podržava mogu se pronaći u odjeljku Podržani Sintetizatori Govora.

1.4. Podrška za brajične retke

Za korisnike koji posjeduju brajični redak, NVDA može prikazivati svoje informacije na Brailleovom pismu. Za više informacija o podržanim brajevim retcima pogledajte odjeljak Podržani Brajevi Retci.

NVDA podržava brajične kodove za mnoge jezike, uključujući kratkopis, puno pismo i kompjutorsku brajicu Za mnoge jezike.

3. Preuzimanje i postavljanje NVDA

Ako još niste preuzeli svoju kopiju NVDA, možete ju preuzeti sa www.nvaccess.org.

Pronađite odjeljak Preuzimanje i tamo ćete pronaći najnoviju verziju NVDA.

Pokrenete li datoteku koju ste upravo preuzeli, pokrenuti će se privremena kopija NVDA. Tada ćete biti pitani želite li instalirati NVDA, napraviti prijenosnu kopiju ili samo nastaviti koristiti privremenu kopiju.

Planirate li stalno koristiti NVDA na ovom računalu, poželjet ćete instalirati NVDA. Instaliranje NVDA će omogućiti dodatne funkcije kao što su to automatsko pokretanje poslije prijave, mogućnost čitanja windows-ovog zaslona za prijavu i windows osigurane zaslone (što nije moguće izvesti sa prijenosnom i privremenom kopijom) i izrada prečaca u izborniku start i na radnoj površini. Instalirana kopija može napraviti prijenosnu kopiju iz same sebe u bilo kojem trenutku.

Ako želite nositi NVDA sa sobom na usb sticku ili na drugom mediju koji podržava zapisivanje, tada trebate izabrati izradu prijenosne kopije. Prijenosna kopija također može instalirati samu sebe kasnije na bilo kojem računalu. međutim, ako želite kopirati NVDA na Medij koji je samo za čitanje kao što je to CD, trebate samo kopirati instalacijsku datoteku koju ste preuzeli. U ovom trenutku pokretanje prijenosne verzije sa medija koji su samo za čitanje nije podržano.

Korištenje privremene kopije također je moguće (npr. za potrebe prezentacije), mada pokretanje NVDA kao privremene kopije može oduzimati vrijeme.

3.1. Ogranićenja prijenosne i privremene kopije NVDA

Osim nemogućnosti automatskog pokretanja tokom i/ili poslije prijave, prijenosne i privremene kopije NVDA također imaju slijedeća ogranićenja:

  • Nemogućnost korištenja aplikacija koje zahtjevaju administratorska prava, ako naravno sam NVDA nije pokrenut sa istim pravima (ne preporuča se).
  • nemogućnost čitanja zaslona kontrole korisničkog računa (UAC) kada se pokušava pokrenuti aplikacija sa administratorskim pravima.
  • Windows 8: nemogućnost podrške unosa za ekrane osjetljive na dodir.
  • Windows 8: Nemogućnost omogućavanja značajki kao što su to način pregleda i izgovor upisanih znakova u aplikacijama kupljenih iz windows prodavaonice.

3.2. Instaliranje NVDA

Ako instalirate NVDA izravno pokrečući preuzeti instalacijski paket, pritisnite gumb instaliraj NVDA. Ako ste već zatvorili ovaj dijaloški okvir ili želite instalirati iz prijenosne kopije, molimo odaberite instaliraj NVDA stavku izbornika koju možete pronaći U podizborniku Alati u NVDA izborniku.

Instalacijski dijaloški okvir koji će se prikazati potvrditi će želite li instalirati NVDA i biti ćete obavješteni hoće li instalacija nadograditi vašu trenutnu kopiju NVDA. Kada pritisnete gumb Nastavi, započeti ćete instalirati NVDA. Također postoji nekoliko opcija u ovom dijalogu koje su objašnjene ispod. Jedanput kada instalacija završi, pojavit će se poruka da je instalacija uspješna. Kada pritisnete U redu, pokrenuti ćete novoinstaliranu kopiju.

3.2.1. Pokreni na windows zaslonu za prijavu

Ova opcija vam omogućava hoće li ili neće NVDA biti pokrenut na windows zaslonu za prijavu, prije nego što upišete lozinku. Ovo također uključuje kontrolu korisničkog računa i ostale sigurnosne zaslone.

3.2.2. Napravi prečac na radnoj površini (ctrl+alt+n)

Ova opcija omogućuje vam odabir između toga hoće li ili neće NVDA napraviti prečac na radnoj povvršini za njegovo pokretanje. Ako je napravljen, prečici na radnoj površini bit će dodjeljena tipkovnička kratica control+alt+n, omogučavajući vam pokretanje NVDA u bilo koje vrijeme.

3.2.3. Kopiraj prijenosnu konfiguraciju na ovaj korisnički račun

Ova opcija vam omogućuje odabir između toga hoće li ili neće NVDA kopirati trenutnu konfiguraciju iz prijenosne kopije na trenutno prijavljeni korisnički račun, za trenutno instaliranu kopiju NVDA. Ova opcija neće kopirati konfiguraciju za bilo kojeg drugog korisnika na tom sustavu niti za konfiguraciju koja se koristi na windowsovom zaslonu za prijavu i drugim sigurnosnim zaslonima. Ova opcija je dostupna jedino kada se NVDA instalira iz prijenosne kopije, ne u trenutku kada se nvda instalira iz instalacijske datoteke koja je preuzeta sa interneta..

3.3. Izrada prijenosne kopije

Ako izrađujete prijenosnu kopiju NVDA iz preuzetog instalacijskog paketa, jednostavno pritisnite gumb napravi prijenosnu kopiju. Ako ste već zatvorili dijaloški okvir ili već imate instaliranNVDA, izaberite stavku izradi prijenosnu kopiju koju možete pronaći u NVDA izborniku.

Dijaloški okvir koji se prikazuje omogućuje vam odabir gdje će prijenosna kopija biti napravljena. To može biti mapa na vašem tvrdom disku ili lokacija na vašem usb sticku ili drugi prijenosni uređaj. Također, postoji opcija koja omogućuje kopiranje konfiguracije trenutno prijavljenog korisnika za korištenje u prijenosnoj kopiji. Ova je opcija dostupna samo onda kada se prijenosna kopija izrađuje iz pozicije instalirane kopije, ali ne i kada se izrađuje iz instalacijske datoteke preuzete s interneta. Kada pritisnete nastavi, započet će proces izrade prijenosne kopije. Kada proces izrade završi, prikazat će se poruka da je izrada prijenosne kopije uspješna. Pritisnite U redu kako bi ste zatvorili ovaj dijaloški okvir.

4. Početak korištenja NVDA

4.1. Pokretanje NVDA

Ako ste instalirali NVDA pomoću instalacijskog programa, onda je pokretanje NVDA jednostavno kao i pritisak control+alt+n, Ili odabirom NVDA iz NVDA izbornika U podizborniku programi u start izborniku. Dodatno, možete upisati "nvda", u dijaloški okvir pokreni i pritisnuti enter kako biste pokrenuli NVDA. Možete također dodati nekoliko opcija naredbenog retka koje vam omogućuju ponovno pokretanje NVDA (-r), izlaz (-q), onemogućavanje dodataka (--disable-addons), itd.

Kako biste pokrenuli prijenosnu inačicu, uđite u mapu u kojoj ste raspakirali NVDA, i pritisnite enter ili dva puta kliknite na datoteku nvda.exe.

Kada se NVDA pokreće, prvo ćete čuti seriju uzlaznih tonova (koji vam govore da se NVDA učitava). Ovisno o tome koliko je vaše računalo brzo, ili ako pokrećete NVDA sa usb ključa ili sa drugog sporog medija, pokretanje može potrajati. Ako učitavanje uzima dodatno vrijeme, NVDA bi trebao izgovoriti "NVDA se učitava. Molimo pričekajte..."

Ako ne čujete ništa od navedenog, ili čujete windows zvuk za grešku, ili silaznu seriju tonova, onda to znači da postoji greška u NVDA, i možda ćete trebati prijaviti grešku razvojnom timu. Molimo pogledajte web stranicu NVDA za više informacija kako to učiniti.

Kada prvi puta pokrenete NVDA, otvorit će se dijaloški okvir dobrodošlice koji vam daje neke osnovne informacije o NVDA modifikacijskoj tipci i NVDA izborniku. (Molimo pogledajte daljnje sekcije o ovim temama.) Dijaloški okvir također sadrži tri potvrdna okvira. Prvi potvrdni okvir daje vam mogućnost odabira hoćete li ili nećete koristiti Caps lock kao modifikacijsku tipku. Drugi određuje hoće li se NVDA pokretati prilikom prijave u Windows i dostupan je samo u instaliranim kopijama NVDA. Treći vam omogućuje odabir između toga hoće li se ili neće ovaj dijalog prikazivati prilikom pokretanja NVDA.

4.2. O tipkovničkim prečacima za NVDA

4.2.1. NVDA modifikacijska tipka

Većina specifičnih NVDA tipkovničkih kratica sastoji se od pritiska određene tipke koja se naziva nvda modifikacijska tipka u sprezi sa jednom ili više tipaka. Značajne iznimke su prečaci za pregled teksta za izgled tipkovnice za stolna računala koje koriste numerički dio tipkovnice od strane samih sebe, ali također postoje i druge iznimke.

NVDA može biti konfiguriran tako da Numerički Insert, Prošireni Insert i/ili caps lock može biti korišten kao NVDA modifikacijska tipka. Podrazumijevano, obje tipke numerički insert i prošireni insert Su podešeni kao NVDA modifikacijske tipke.

Ako želite prouzročiti da se NVDA modifikacijske tipke ponašaju na način kao da NVDA nije pokrenut (npr. želite uključiti velika slova Kada ste postavili capslock kao modifikacijsku tipku), možete pritisnuti tipku dva puta brzo.

4.2.2. Izgledi tipkovnice

NVDA trenutno dolazi sa dva seta tipkovničkih prečaca (znani kao izgledi tipkovnice): izgled tipkovnice za stolna računala i izgled tipkovnica za prijenosna računala. Podrazumijevano, NVDA koristi izgled tipkovnice za stolna računala, ali se možete prebaciti na izgled tipkovnice za prijenosna računala u dijaloškom okviru postavke tipkovnice, koje možete pronaći u podizborniku postavke u NVDA izborniku.

izgled tipkovnice za stolna računala maksimalno iskorištava numeričku tastaturu (bez uključenog numeričkog bloka). Iako neka prijenosna računala nemaju numeričku tipkovnicu, neka prijenosna računala mogu emulirati numeričku tipkovnicu pritiskom tipke FN i pritiščući slova i brojeve koji se nalaze sa desne strane na tipkovnici (7, 8, 9, u, i, o, j, k, l, itd.). Ako vaše prijenosno računalo ne može to izvršiti ili vam ne omogućuje isključivanje numeričkog bloka, možda umjesto toga se želite prebaciti na raspored tipkovnice za prijenosna računala.

4.3. NVDA dodirne geste

Ako pokrećete NVDA na uređaju koji ima zaslon osjetljiv na dodir i koji pokreće windows 8 ili viši, možete također upravljati NVDA izravno pomoću zaslona osjetljivog na dodir. Kada je NVDA pokrenut, Svaki dodir sa ekrana osjetljivog na dodir bit će prosljeđen izravno u NVDA. Dakle, akcije koje mogu biti učinjene normalno bez korištenja NVDA neće raditi.

4.3.1. Istraživanje zaslona

Najosnovnija stvar koju možete učiniti sa zaslona osjetljivog na dodir je izvještavanje o kontroli ili tekstu na bilo kojem dijelu zaslona. Kako biste to učinili, postavite prst na bilo koje mjesto na zaslonu. Također možete držati prst na zaslonu i pomicati ga u krug kako bi ste pročitali ostale i tekst po kojima se premještate vašim prstom.

4.3.2. dodirne geste

Kada će NVDA komande biti opisivane kasnije u ovom korisničkom vodiću, one mogu također imati dodirnu gestu koja se može koristiti za aktiviranje te komande pomoću ekrana osjetljivog na dodir. Slijedi nekoliko uputa kako izvesti određene dodirne geste.

Dodiri

Dodirujte brzo zaslon za jednim ili više prstiju.

Dodirivanje samo sa jednim prstom jednostavno je poznato kao dodir. Dodirivanje sa dva prsta je poznato kao dvoprstni dodir i tako dalje.

Ako je isti dodir izveden jedan ili više puta ponovo brzo, NVDA će to shvatiti kao višedodirnu gestu. Dvostruki dodir prouzrukuje dvostruko dodirivanje nekog elementa na zaslonu. dodirivanje 3 puta će prouzročiti trostruki dodir i tako dalje. Naravno, ove višedodirne geste prepoznaju koliko sse prstiju koristi, dakoe, moguće je imati geste kao što su to na primjer trostruki dodir sa dva prsta, dodir sa četiri prsta, itd.

Klizanje

NVDA također može biti konfiguriran da obavješta va o tipu objekta ispod miša kako se kreće (npr. popis, gumb, etc.). To može biti korisno potpuno slijepim korisnicima, jer ponekada tekst nije dovoljan.

NVDA korisnicima način da razumiju gdje se miš nalazi u vezi sa dimenzijama zaslona reproducirajući zvučne kordinate kao zvučne signale. čim je miš na zaslonu viši, visina zvučnih signala je veća. Ako je više lijevo ili desno miš na zaslonu, više lijevo ili desno zvuk će biti reproduciran (ako uzmemo u obzir da korisnik ima stereo zvučnike ili slušalice).

Po podrazumjevanom ove značajke nisu uključene u postavkama NVDA. Ako želite imati koristi od tih značajki, možete ih podesiti the postavkama miša , koje se mogu pronaći u postavkama koje se nalaze u NVDA izborniku.

8. Podešavanje NVDA

8.1. Postavke

Većina postavki NVDA mogu se promijeniti koristeći dijaloške okvire kojima se može pristupiti preko podizbornika postavke NVDA izbornika. U svim dijaloškim okvirima postavki NVDA, pritisnite gumb u redu kako biste primjenili promjene koje ste napravili. Kako biste poništili promjene, pritisnite gumb odustani ili escape tipku na tipkovnici. Većina se postavki također mogu mijenjati koristeći tipkovničke prečace, koji su navedeni gdje je to relevantno u u odjeljcima ispod.

8.1.1. opće postavke (NVDA+control+g)

Dijaloški okvir opće postavke možete pronaći u podizborniku postavke. Dijaloški okvir sadrži slijedeće opcije:

Jezik

Ovo je odabirni okvir koji vam omogućuje odabir jezika na kojem će sve poruke NVDA i sućelje biti prikazani. Postoji puno jezika, međutim podrazumijevana opcija je "Korisnički zadano, Windows". Kada je ova opcija aktivna, NVDA koristi podrazumijevani jezik sustava.

Imajte na umu da NVDA mora biti ponovno pokrenut kada mijenjate jezik. NVDA će vas pitati želite li ponovno pokrenuti NVDA, ako želite promijeniti odabir. Pritisnite u redu, i NVDA će ponovno biti pokrenut.

Spremi konfiguraciju pri izlazu

Ova je opcija potvrdni okvir koji, kada je odabran, prouzrokuje da NVDA sprema konfiguraciju kada izađete iz NVDA.

Prikaži opcije za iskljućivanje kada se NVDA iskljućuje

Ova je opcija potvrdni okvir koji vam omogućuje hoće li se ili neće prikazati dijaloški okvir u kojem vas se pita koju akciju želite izvesti. Kada je odabran, kada pokušate iskljućiti NVDA prikazat će se dijaloški okvir u kojem ćete biti pitani želite li iskljućiti, ponovno pokrenuti ili ponovno pokrenuti NVDA bez omogućenih dodataka. Kada nije odabran ovaj potvrdni okvir, NVDA će se odmah isključiti.

Reproduciraj zvukove kada se NVDA uključuje ili isključuje

Ova opcija je potvrdni okvir koji, kada je odabran, prouzrokuje da NVDA reproducira zvukove kada se pokrene ili iskljući.

Razina loga

Ovo je odabirni okvir koji vam omogućuje odabir koliko će log poruka NVDA prikazivati u pregledniku loga kada je pokrenut. Zapravo, korisnici ovo ne bi trebali dirati jer nije baš sve zapisano u log. Međutim ako želite dati više informacija uz prijavu greške, Onda ovo može biti korisna opcija.

automatski pokreni NVDA kada se prijavim u sustav windows

Ako je ova opcija omogućena, NVDA će se automatski pokrenuti kada se prijavite u windows. Ova opcija je dostupna samo u instaliranim inačicama NVDA.

koristi NVDA na windows zaslonu za prijavu (zahtjeva administratorska prava)

Ako se prijavljujete u Windows upisujući svoje korisničko ime i lozinku, Onda će ova opcija prouzročiti automatsko pokretanje NVDA na zaslonu za prijavu kada se operativni sustav Windows pokreće. Ova je opcija dostupna samo u instaliranim inačicama NVDA.

Koristi trenutne postavke na zaslonu za prijavu i drugim sigurnosnim zaslonima

Kada je ovaj gumb pritisnut, NVDA postavke će biti kopirane u NVDA mapu za konfiguracije sustava, tako da će se te postavke koristiti na windowsovom zaslonu za prijavu, kontroli korisničkog računa (UAC) i na drugim sigurnosnim zaslonima windowsa. Kako biste se uvjerili da su sve vaše postavke kopirane, prvo spremite vašu konfiguraciju pomoću prečice control+NVDA+c ili pomoću opcije spremi konfiguraciju u NVDA izborniku. Ova je opcija dostupna samo u instaliranim inačicama NVDA.

automatski provjeri postojanje nove verzije NVDA

Ako je ova opcija ukljućena, NVDA će automatski provjeriti za nadogradnje samoga sebe i obavjestiti vas da je nadogradnja dostupna. Također možete ručno provjeravati ima li nova verzija kada odaberete provjeri postojanje nadogradnje u podizborniku pomoć u NVDA izborniku.

8.1.2. Odabir govorne jedinice (NVDA+control+s)

Jedamput kada ste odabrali govornu jedinicu koju želite koristiti, možete pritisnuti U redu i NVDA će učitati tu govornu jedinicu. Govorna jedinica

Ova opcija omogučava odabir govorne jedinice koju želite da NVDA koristi pri izgovoru teksta.

Kako biste vidjeli listu govornih jedinica koje podržava NVDA, molimo pogledajte odjeljak podržane govorne jedinice.

Jedna posebna stavka koja će se uvjek pojavljivati u ovom popisu je "bez govora", koja vam omogućuje korištenje NVDA bez govornog izlaza. To može biti korisno za nekoga tko želi koristiti NVDA samo uz pomoć brajičnog retka, ili konačno za razvojne programere koji žele koristiti preglednik govora.

Izlazni uređaj

Ova opcija omogućuje odabir zvučne kartice preko koje će NVDA proslijeđivati izlaznu govornu informaciju prema korisniku.

8.1.3. Glasovne postavke (NVDA+control+v)

Dijaloški okvir postavke glasa, kojeg možete pronaći u izborniku postavke, sadrži opcije koje vam omogućuju promjenu parametara govora. Za brže mijenjanje glasovnih parametara sa bilo kojeg mjesta pogledajte sekciju postavke sintetizatora govora u obliku prstena.

Dijaloški okvir glasovne postavke sadrži slijedeće opcije:

Glas

Ovo je prva opcija na koju ćete naići, a koja popisuje sve instalirane glasove u sustavu. Možete pritisnuti strelice gore i dolje kako biste preslušali raznolike izbore glasova na vašem sustavu. Lijeva i gornja strelica vam daje mogućnost kretanja prema gore u popisu, dok desna i doljnja strelica premješta vas prema dolje u popisu.

Vrsta

Ako koristite Espeak govornu jedinicu koja dolazi sa NVDA, ovo je odabirni okvir koji vam omogućuje označavanje vrste glasa koju će sintetizator govora koristiti za izgovor. Espeakove varijante su poput glasova, jer espeakovim glasovima daju malčice drugaćije atribute. Neke će varijante zvučati kao muški glas, neke kao ženski, neke čak i kao žaba.

Brzina

Ova opcija vam omogućuje promjenu brzine glasa. Ovo je klizač koji ima vrijednosti od 0 do 100, (0 predstavlja najsporiju brzinu, 100 predstavlja najbržu brzinu).

Visina

Ova vam opcija omogućuje promjenu visinu trenutnog glasa. Ovo je klizač koji ide od 0 do 100, ( 0 kao najniža visina a 100 kao najviša visina).

Glasnoća

Ova je opcija klizač koji ide od 0 do 100, (0 kao najtiša vrijednost dok je 100 najglasnija).

Modulacija

Ova je opcija klizač koji vam omogućuje odabir, odnosno podešavanje modulacije sintetizatora (podizanje i spuštanje visine pri čitanju) koju će sintetizator koristiti. (Trenutno samo espeak podržava ovu mogućnost).

Automatska promjena jezika

Ovaj odabirni okvir omogućuje vam odabir ukljućenja / iskljućenja automatskog prepoznavanja jezika ako je u dokumentu takvo što označeno npr, na web stranicama ili u wordovim dokumentima. Ova je opcija po podrazumijevanom ukljućena. Trenutno samo Espeak podržava ovu opciju.

Automatsko prebacivanje dijalekata

Ovaj potvrdni okvir vam omogućuje hoće li ili neće biti otkrivene promjene dijalekta pri čitanju, to je bolje nego samo prebacivanje jezika. Na primjer, ako čitate uz pomoć glasa za Američki engleski ali je u dokumentu postavljen dijalekat Engleskog Velike britanije, onda će sintetizator prebacivati između naglasaka ako je ova opcija označena. Ova je opcija po podrazumijevanom onemogućena.

Razina čitanja interpunkcije

Prečac: NVDA+p

Ova vam opcija omogućuje određivanje kolićine interpunkcijskih znakova i drugih simbola koji trebaju biti izgovoreni kao riječi. Na primjer, Kada je ova opcija postavljena na način da se izgovaraju svi simboli, Svi će simboli biti izgovoreni kao riječi. Ova se opcija primjenjuje na sve govorne jedinice, ne samo na trenutno aktivnu.

Postotak visine glasa kod velikih slova

Ovo polje za uređivanje omogućuje vam promjenu vrijednosti visine koju će govorna jedinica koristiti pri izgovaranju velikih slova. Ova je vrijednost oblikovana kao postotak, u kojoj negativna vrijednost snižava visinu dok ju pozitivna vrijednost bez predznaka minus povisuje. Kada ne želite promjenu visine koristit ćete 0 kao vrijednost.

Izgovori "veliko" prije velikih slova

Ova je opcija potvrdni okvir koji, kada je odabran, prouzrokuje da NVDA izgovara riječ "veliko", prije svakog velikog slova na primjer kada slovkamo tekst. Uobičajeno, NVDA malo povisuje visinu glasa za bilo koje veliko slovo, ali, neke govorne jedinice neće podržavati takav način izgovora velikih slova, stoga ova opcija može biti od koristi.

Zvučni signal za velika slova

Ako je ovaj potvrdni okvir odabran, NVDA će proizvesti zvučni signal kada se nađe na velikom slovu. Kao i potvrdni okvir "Izgovori veliko prije velikih slova", Ovo je korisno koristiti uz govorne jedinice koje ne mogu povisiti glas kod velikih slova.

Neke se riječi sastoje od samo jednog znaka, ali je izgovor različit ovisno o tome jeli znak izgovoren kao jedan znak (kao kada se sriče) ili kao riječ. 8.1.10. Input Composition Settings

The Input Composition Settings dialog can be found under the Preferences menu. This dialog allows you to control how NVDA reports the input of Asian characters, such as with IME or Text Service input methods. 8.1.13. Speech dictionaries

The speech dictionaries menu (found in the Preferences menu) contains dialogs that allow you to manage the way NVDA pronounces particular words or phrases. There are currently three different types of speech dictionaries. They are:

  • Default: rules in this dictionary affect all speech in NVDA.
  • Voice: rules in this dictionary affect speech for the synthesizer voice currently being used.
  • Temporary: rules in this dictionary affect all speech in NVDA, but only for the current session. These rules are temporary and will be lost if NVDA is restarted.

All dictionary dialogs contain a list of rules which will be used for processing the speech. The dialog also contains Add, Edit and Remove buttons.

To add a new rule to the dictionary, press the Add button, and fill in the fields in the dialog box that appears and then press Ok. You will then see your new rule in the list of rules. However to make sure your rule is actually saved, make sure to press Ok to exit the dictionary dialog all together once you have finished adding/editing rules.

The rules for NVDA's speech dictionaries allow you to change one string of characters into another. A simple example would be that you want to have NVDA say the word frog each time it is supposed to say the word bird. In the Add rule dialog, the easiest way to do this is to type the word bird in the Pattern field, and the word frog in the Replacement field. You may also want to type a description of the rule in the Comment field (something like: changes bird to frog).

NVDA's speech dictionaries however are much more powerful than simple word replacement. The Add rule dialog also contains a checkbox to say whether or not you want the rule to be case sensitive (meaning that NVDA should care whether the characters are uppercase or lowercase. NVDA ignores case by default).

Finally, a set of radio buttons allows you to tell NVDA whether your pattern should match anywhere, should only match if it is a complete word or should be treated as a "Regular expression". Setting the pattern to match as a whole word means that the replacement will only be made if the pattern does not occur as part of a larger word; i.e. a character other than an alphanumeric character or an underscore (or no character at all) comes both immediately before and after the pattern. Thus, using the earlier example of replacing the word "bird" with "frog", if you were to make this a whole word replacement, it would not match "birds" or "bluebird".

A regular expression is a pattern containing special symbols that allow you to match on more than one character at a time, or match on just numbers, or just letters, as a few examples. Regular expressions are not covered in this user guide, but there are many tutorials on the web which can provide you with more information.

8.3. Configuration Profiles

Sometimes, you may wish to have different settings for different situations. For example, you may wish to have reporting of indentation enabled while you are editing or reporting of font attributes enabled while you are proofreading. NVDA allows you to do this using configuration profiles.

A configuration profile contains only those settings which are changed while the profile is being edited. Most settings can be changed in configuration profiles except for those in the General Settings dialog, which apply to the entirety of NVDA.

Configuration profiles can be manually activated. They can also be activated automatically due to triggers such as switching to a particular application.

8.3.1. Basic Management

You manage configuration profiles by selecting "Configuration profiles" in the NVDA menu. You can also do this using a key command:

  • NVDA+control+p: Show the Configuration Profiles dialog.

The first control in this dialog is the profile list from which you can select one of the available profiles. When you open the dialog, the profile you are currently editing is selected. Additional information is also shown for active profiles, indicating whether they are manually activated, triggered and/or being edited.

To rename or delete a profile, press the Rename or Delete buttons, respectively.

Press the Close button to close the dialog.

8.3.2. Creating a Profile

To create a profile, press the New button.

In the New Profile dialog, you can enter a name for the profile. You can also select how this profile should be used. If you only want to use this profile manually, select Manual activation, which is the default. Otherwise, select a trigger which should automatically activate this profile. For convenience, if you haven't entered a name for the profile, selecting a trigger will fill in the name accordingly. See below for more information about triggers.

Pressing OK will create the profile and close the Configuration Profiles dialog so you can edit it.

8.3.3. Manual Activation

You can manually activate a profile by selecting a profile and pressing the Manual activate button. Once activated, other profiles can still be activated due to triggers, but any settings in the manually activated profile will override them. For example, if a profile is triggered for the current application and reporting of links is enabled in that profile but disabled it in the manually activated profile, links will not be reported. However, if you have changed the voice in the triggered profile but have never changed it in the manually activated profile, the voice from the triggered profile will be used. Any settings you change will be saved in the manually activated profile. To deactivate a manually activated profile, select it in the Configuration Profiles dialog and press the Manual deactivate button.

8.3.4. Triggers

Pressing the Triggers button in the Configuration Profiles dialog allows you to change the profiles which should be automatically activated for various triggers.

The Triggers list shows the available triggers, which are as follows:

  • Current application: Triggered when you switch to the current application.
  • Say all: Triggered while reading with the say all command.

To change the profile which should be automatically activated for a trigger, select the trigger and then select the desired profile from the Profile list. You can select (normal configuration) if you don't want a profile to be used.

Press the Close button to return to the Configuration Profiles dialog.

8.3.5. Editing a Profile

If you have manually activated a profile, any settings you change will be saved to that profile. Otherwise, any settings you change will be saved to the most recently triggered profile. For example, if you have associated a profile with the Notepad application and you switch to Notepad, any changed settings will be saved to that profile. Finally, if there is neither a manually activated nor a triggered profile, any settings you change will be saved to your normal configuration.

To edit the profile associated with say all, you must manually activate that profile.

8.3.6. Temporarily Disabling Triggers

Sometimes, it is useful to temporarily disable all triggers. For example, you might wish to edit a manually activated profile or your normal configuration without triggered profiles interfering. You can do this by checking the Temporarily disable all triggers checkbox in the Configuration Profiles dialog.

8.4. Location of Configuration files

Portable versions of NVDA store all settings, custom appModules and custom drivers in a directory called userConfig, found in the NVDA directory.

Installed versions of NVDA store all settings, custom appModules and custom drivers in a special NVDA directory located in your Windows user profile. This means that each user on the system can have their own NVDA settings. To get to your settings directory for an installed version of NVDA, on the start menu you can go to programs -> NVDA -> explore user configuration directory.

Settings for NVDA when running on the logon or UAC screens are stored in the systemConfig directory in NVDA's installation directory. Usually, this configuration should not be touched. To change NVDA's configuration on the logon/UAC screens, configure NVDA as you wish while logged into Windows, save the configuration, and then press the "Use currently saved settings on the logon and other secure screens" button in the General Settings dialog.

The log viewer, found under Tools in the NVDA menu, allows you to view all the logging output that has occured up until now from when you last started NVDA.

Apart from reading the content, you can also Save a copy of the log file, or refresh the viewer so that it shows the most recent output since the Log viewer was opened. These actions are available under the viewer's Log menu.

9.2. Speech Viewer

For sighted software developers or people demoing NVDA to sighted audiences, a floating window is available that allows you to view all the text that NVDA is currently speaking.

To enable the speech viewer, check the "Speech Viewer" menu item under Tools in the NVDA menu. Uncheck the menu item to disable it.

While the speech viewer is enabled, it constantly updates to show you the most current text being spoken. However, if you click or focus inside the viewer, NVDA will temporarily stop updating the text, so that you are able to easily select or copy the existing content.

9.3. Add-ons Manager

The Add-ons Manager, accessed by selecting Manage add-ons under Tools in the NVDA menu, allows you to install and uninstall add-on packages for NVDA. These packages are provided by the community and contain custom code that may add or change features in NVDA or even provide support for extra Braille displays or speech synthesizers.

The Add-ons Manager contains a list that displays all the add-ons currently installed in your NVDA user configuration. A package name, version and author are shown for each add-on, though further information such as a description and URL can be viewed by selecting the add-on and pressing the About add-on button. If there is help available for the selected add-on, you can access it by pressing the Add-on help button.

To browse and download available add-ons online, press the Get add-ons button. This button opens the NVDA Add-ons page. If NVDA is installed on your system, you can open the add-on directly from the browser to begin the installation process as described below. Otherwise, save the add-on package and follow the instructions below.

To install an Add-on you previously obtained, press the Install button. This will allow you to browse for an add-on package (.nvda-addon file) somewhere on your computer or on a network. Once you press Open, the installation process will begin.

When an add-on is being installed, NVDA will first ask you to confirm that you really wish to install the add-on. As the functionality of add-ons is unrestricted inside NVDA, which in theory could include accessing your personal data or even the entire system if NVDA is an installed copy, it is very important to only install add-ons from sources you trust. Once the add-on is installed, NVDA must be restarted for the add-on to start running. Until you do, a status of "install" will show for that add-on in the list.

To remove an add-on, select the add-on from the list and press the Remove button. NVDA will ask if you really wish to do this. As with installing, NVDA must be restarted for the add-on to be fully removed. Until you do, a status of "remove" will be shown for that add-on in the list.

The manager also has a Close button to close the dialog. If you have installed or removed add-ons, NVDA will first ask you if you wish to restart so that your changes can take effect.

In the past it has been possible to extend NVDA's functionality by copying individual plugins and drivers in to your NVDA user Configuration directory. Although this version of NVDA may still load them, they will not be shown in the Add-on Manager. It is best to remove these files from your configuration and install the appropriate add-on if one is available.

9.4. Python Console

The NVDA Python console, found under Tools in the NVDA menu, is a development tool which is useful for debugging, general inspection of NVDA internals or inspection of the accessibility hierarchy of an application. For more information, please see the Developer Guide available from the Development section of the NVDA web site.

9.5. Reload plugins

This item, once activated, reloads app modules and global plugins without restarting NVDA, which can be useful for developers.

The eSpeak synthesizer is built directly into NVDA and does not require any other special drivers or components to be installed. NVDA starts using eSpeak by default. As this synthesizer is built into NVDA, this is a great choice for when running NVDA off a USB thumb drive on other systems.

SAPI 4 is an older Microsoft standard for software speech synthesizers. When using this synthesizer with NVDA, the available voices (accessed from the Voice Settings dialog or by the Synth Settings Ring) will contain all the voices from all the installed SAPI 4 engines found on your system.

If you have SAPI 4 voices installed but the synthesizer does not appear in NVDA's list of synthesizers, please install the SAPI 4.0 runtime binaries, available at http://activex.microsoft.com/activex/controls/sapi/spchapi.exe.

13. Advanced Topics

13.1. Command Line Options

NVDA can accept one or more additional options when it starts which alter its behavior. You can pass as many options as you need. These options can be passed when starting from a shortcut (in the shortcut properties), from the Run dialog (Start Menu -> Run or Windows+r) or from a Windows command console. Options should be separated from the name of NVDA's executable file and from other options by spaces. For example, the Desktop shortcut that NVDA creates during installation has the -r option, which tells NVDA to close the currently running copy before starting the new one. Another useful option is --disable-addons, which tells NVDA to suspend all running add-ons. This allows you to determine whether a problem is caused by an add-on and to recover from serious problems caused by add-ons.

As an example, you can exit the currently running copy of NVDA by entering the following in the Run dialog:

nvda -q

Some of the command line options have a short and a long version, while some of them have only a long version. For those which have a short version, you can combine them like this:

nvda -rm This will exit the currently running copy of NVDA and will start a new copy with startup sounds disabled, etc.
nvda -rm --disable-addons Same as above, but with add-ons disabled

Some of the command line options accept additional parameters; e.g. how detailed the logging should be or the path to the user configuration directory. Those parameters should be placed after the option, separated from the option by a space when using the short version or an equals sign (=) when using the long version; e.g.:

nvda -l 10 Tells NVDA to start with log level set to debug
nvda --log-file=c:\nvda.log Tells NVDA to write its log to c:\nvda.log
nvda --log-level=20 -f c:\nvda.log Tels NVDA to start with log level set to info and to write its log to c:\nvda.log

13.2. Advanced Customization of Symbol Pronunciation

It is possible to customize the pronunciation of punctuation and other symbols beyond what can be done using the Punctuation/symbol pronunciation dialog. For example, you can specify whether the raw symbol should be sent to the synthesizer (e.g. to cause a pause or change in inflection) and you can add custom symbols.

To do this, you must edit the symbol pronunciation information file in your NVDA user configuration directory. The file is called symbols-xx.dic, where xx is the language code. The format of this file is documented in the Symbol Pronunciation section of the NVDA Developer Guide, which is available from the Development section of the NVDA web site. However, it is not possible for users to define complex symbols.
