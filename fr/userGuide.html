1. Introduction

NonVisual Desktop Access (NVDA) est une revue d'écran gratuite et open-source pour le système d'exploitation Microsoft Windows. En donnant des informations via une voix synthétique et le Braille, il permet aux personnes aveugles ou malvoyantes d'accéder à un ordinateur sans coût additionnel par rapport à une personne voyante. NVDA est développé par NV Access, avec des contributions de la communauté.

1.1. Caractéristiques générales

En restituant l'information par synthèse vocale ou en Braille, NVDA permet aux personnes aveugles et mal-voyantes d'utiliser le système d'exploitation Windows ainsi que beaucoup d'applications disponibles dans cet environnement.

Ses points forts sont :

  • La prise en charge d'applications grand-public telles que des navigateurs Web, clients e-mail, programmes de tchat sur Internet et suites bureautiques
  • Un synthétiseur intégré prenant en charge plus de 43 langues
  • l'annonce, quand c'est possible, d'informations de formatage du texte telles que le nom et la taille de la police, le style et les fautes d'orthographe
  • L'annonce automatique du texte sous la souris et, en option, l'indication sonore de la position de la souris
  • La prise en charge de nombreux afficheurs braille, incluant la saisie du braille informatique pour les afficheurs disposant d'un clavier braille
  • La possibilité de s'exécuter entièrement depuis une clé USB ou tout autre média portable sans avoir à installer
  • Un installateur parlant facile à utiliser
  • Une traduction dans 44 langues
  • La prise en charge des environnements Windows modernes, incluant les versions 32 et 64 bits
  • L'accès à l'écran de connexion à Windows et aux autres écrans sécurisés
  • Prise en charge des interfaces d'accessibilités communes telles que Microsoft Active Accessibility, Java Access Bridge, IAccessible2 et UI Automation (UI Automation n'est supporté qu'à partir de Windows 7)
  • Support de l'invite de commandes Windows et des applications en mode console.

1.2. Internationalisation

Il est important que chacun, partout dans le monde, quelle que soit sa langue, ait le même accès aux technologies. A ce jour, NVDA a été traduit en 44 langues : l'Afrikaans, l'Albanais, l'Allemand, l'Arabe, l'Aragonais, le Bulgare, le Chinois Traditionnel et simplifié, le Coréen, le Croate, le Danois, l'Espagnol, l'Éthiopien, le Farsi, le Finnois, le Français, le Galicien, le Grecque, le Géorgien, l'Hébreu, le Hongrois, l'Hindi, l'Irlandais, l'Islandais, l'Italien, le Japonais, le Néerlandais, le népalais, le Norvégien, le Polonais, le Portugais, le Portugais Brésilien, le Roumain, le Russe, le Serbe, le Slovaque, le Slovénien, le Suédois, le Tamoul, le Tchèque, le Thaï, le Turc, l'Ukrainien et le Vietnamien.

1.3. Support des synthèses vocales

En plus de proposer ses messages et son interface en plusieurs langues, NVDA permet également à l'utilisateur de lire les contenus dans n'importe quelle langue, dans la mesure où il possède un synthétiseur parlant cette langue.

NVDA est fourni avec eSpeak, un synthétiseur de parole gratuit, open-source, multilingue.

Pour des informations concernant les autres synthétiseurs de parole pris en charge par NVDA, reportez-vous à la section Synthétiseurs de paroles pris en charge.

1.4. Support du Braille

Pour les utilisateurs possédant un terminal braille, NVDA peut afficher ses informations en Braille. Reportez-vous à la section Terminaux braille pris en charge pour les informations concernant les terminaux braille.

NVDA propose de nombreux codes braille couvrant un grand ensemble de langues. Dans bien des cas, le Braille intégral et abrégé sont disponibles. Pour les utilisateurs francophones, le Braille français unifié est disponible en intégral et abrégé.

2. Configuration système requise

  • Systèmes d'exploitation : Toutes les variantes, 32 ou 64 bits, de Windows XP, Windows Vista, Windows 7 et Windows 8 (y compris les versions Serveur).
  • Mémoire : 256 mo de RAM ou plus.
  • Vitesse processeur : 1.0 ghz ou plus.
  • Environ 50 MO d'espace disque.

3. Se procurer et installer NVDA

Si vous n'avez pas encore une copie de NVDA, vous pouvez la télécharger sur nvda-project.org.

Rendez-vous à la section "Download" où vous trouverez un lien pour le téléchargement de la dernière version de NVDA.

Démarrer le fichier télécharger exécutera une version temporaire de NVDA. Vous aurez en suite le choix entre installer NVDA, créer une copie portable, ou juste continuer à utiliser la version temporaire.

Si vous avez prévu de toujours utiliser NVDA sur cette ordinateur, choisissez installer NVDA. Installer NVDA permettra plus de choses comme son lancement après l'identification, il pourra lire les écran d'identification et de sécurité (ce qu'une version portable ou la version temporaire ne peuvent faire), et il sera accessible depuis des raccourcis dans le menu démarrer ou sur le bureau. La version installée permet aussi de créer une copie portable à tout moment.

Si vous voulez prendre NVDA avec vous sur une clé USB ou un autre support accessible en écriture, vous devriez choisir de créer une copie portable. La copie portable peut aussi être installer sur n'importequel ordinateur à n'importequel moment. Cependant, si vous souhaitez copier NVDA sur un support en lecture seul comme un CD, vous devriez seulement copier le package téléchargé. Démarrer dirrectement la copie portable depuis un support en lecture seule n'est pas supporté pour le moment.

Utiliser la version temporaire de NVDA est aussi une option, par exemple pour une démonstration, car démarrer NVDA dans ce mode à chaque fois peut prendre beaucoup de temps.

3.1. Restrictions des versions portables et temporaires

A part l'impossibilité de démarrer pendant ou après l'ouverture de session, les versions portables et temporaire de NVDA ont également les restrictions suivantes :

  • Impossible d'interagir avec les applications démarrées avec les privilèges administrateur, sauf biensur si NVDA a été démarré avec ces privilèges (non recommandé).
  • Impossible de lire les écrans de contrôle de compte utilisateur (UAC) lorsque l'on tente de démarrer une application avec les privilèges administrateur.
  • Windows 8 : impossible d'utiliser un écran tactile comme support d'entrée.
  • Windows 8 : fonctionnalités impossible comme le mode navigation ou l'annonce des caractères saisis dans les applications du Windows Store.

3.2. Installer NVDA

Si vous installez directement par le package téléchargé, cliquez sur le bouton Installer NVDA sur cet ordinateur. Si vous avez déjà fermé cette boîte de dialogue, ou si vous vvoulez l'installer à partir d'une copie portable, allez dans le menu NVDA, cliquer sur "Outils", et choisissez "installer NVDA".

La boîte de dialogue d'installation qui s'affichera vous demandera de confirmer l'installation, et vous signalera s'il s'agit de la mise à jour d'une installation existante. Cliquer sur le bouton Continuer installera NVDA. Deux options détaillées ci-dessous sont disponibles dans ce dialogue. Dès que l'installation sera terminée, un message s'affichera vous signalant la fin de l'installation. Cliquer sur OK démarrera la version nouvellement installée.

3.2.1. Utiliser NVDA sur l'écran de connexion à Windows

Cette option vous permet de choisir si NVDA doit ou non démarrer automatiquement quand vous ête sur l'écran de connexion, avant que vous ayez saisi un mot de passe. Ceci inclut les contrôle UAC et les écran sécurisés.

3.2.2. Créer une icône sur le bureau et le raccourci clavier (control+alt+n)

Cette option vous permet de choisir si NVDA doit ou non créer un raccourci sur le bureau pour démarrer NVDA. Si créé, ce raccourci aura la touche de raccourci control+alt+n assignée pour démarrer NVDA de n'importe où.

3.2.3. Copier la configuration portable dans le compte de l'utilisateur en cours

Cette option permet de choisir si NVDA doit ou non copier la configuration utilisateur depuis la copie de NVDA en cour d'exécution vers la configuration de l'utilisateur courant, pour la version installée de NVDA. Cela ne copiera la configuration ni pour un autre utilisateur du système, ni pour la configuration du système dans le cas de l'utilisation dans l'identification windows ou les autres écrans sécurisés. Cette option n'est disponible que lorsque vous installez NVDA depuis une copie portable, pas quand vous installez depuis le paquet téléchargé du lanceur.

3.3. Créer une copie portable

Si vous voulez créer votre copie depuis le package téléchargé, cliquez simplement sur le bouton Créer une copie portable. Si vous avez déjà fermé cette boîte de dialogue ou si vous êtes dans une version installée, allez dans le menu NVDA, cliquer sur "Outils", et choisissez "Créer une copie portable".

La boîte de dialogue qui s'affiche vous permet de choisir où créer la copie portable. Ce peut être un répertoire de votre disque dure, un chemin sur une clé USB ou un autre média portable. Vous pouvez choisir si oui ou non NVDA doit copier la configuration NVDA de l'utilisateur connecté pour l'utiliser avec la copie portable nouvellement créée. Cette option n'est disponible que lorsque vous créez une copie portable de NVDA depuis une copie installée, pas quand vous créez une copie portable depuis le paquet téléchargé du lanceur. Cliquer sur continuer créera la copie portable. Dès que la création est terminée, un message s'affiche. Cliquez sur OK pour fermer cette boîte de dialogue.

4. Prendre en main NVDA

4.1. Démarrer NVDA

Si NVDA a été installé à l'aide de l'installateur, alors il suffit, soit de frapper la combinaison control+alt+n, soit de choisir dans le menu Démarrer / Programmes / NVDA / NVDA. Vous pouvez aussi taper NVDA dans le dialogue Exécuter puis appuyer sur Entrée.

Pour démarrer la copie portable, rendez-vous dans le dossier où NVDA a été décompressé, et appuyez sur entrée ou double-cliquez sur nvda.exe.

Au démarrage de NVDA, on entend d'abord une série de sons ascendants (indiquant que NVDA se charge). Suivant la rapidité de votre ordinateur, ou si vous exécutez NVDA à partir d'une clé USB ou d'un autre périphérique lent, le démarrage peut demander un certain temps. Si le démarrage est vraiment lent, vous entendrez le message "Chargement de NVDA, veuillez patienter...".

Si vous n'entendez rien de tout cela, ou si vous entendez le signal sonore d'erreur de Windows, ou une série de sons descendants, cela indique un problème de NVDA et vous pourrez, si vous le voulez, envoyer un rapport d'erreur au développeurs. Rapportez-vous au site Web de NVDA pour les détails de la procédure.

Quand NVDA démarre pour la première fois, vous êtes accueilli par une boîte de dialogue vous donnant des informations de base sur la touche NVDA et le menu NVDA. Veuillez consulter les sections ci-après pour plus d'information sur ces sujets. La boîte de dialogue contient également trois cases à cocher. La première vous permet de choisir si vous voulez utiliser la touche de verrouillage majuscules comme touche NVDA. La deuxième spécifie si NVDA doit démarrer à l'ouverture de votre session et est disponible uniquement sur la version installée. La troisième permet d'indiquer si ce dialogue d'accueil doit apparaître à chaque démarrage.

4.2. À propos des commandes clavier de NVDA

4.2.1. La touche NVDA

La plupart des commandes spécifiques de NVDA se font en appuyant sur la touche NVDA en conjonction avec une ou plusieurs autres touches. Les fonctions de revue de texte qui n'utilisent que le pavé numérique constituent une exception.

NVDA peut être configuré pour que les touches Insert du pavé numérique, Insert du clavier étendu ou verrouillage majuscules puissent être utilisés comme touche NVDA.

Par défaut, les touches insert du pavé numérique et du clavier étendu sont définies comme touche NVDA.

Si vous voulez utiliser l'une des touches NVDA dans sa fonction originale (par exemple si vous voulez utiliser verrouillage majuscules pour verrouiller les majuscules après l'avoir définie comme touche NVDA) vous devrez appuyer deux fois rapidement sur la touche.

4.2.2. Les configurations Clavier

NVDA comporte actuellement deux configurations clavier : une pour les ordinateurs de bureau (Desktop) et une pour les ordinateurs portables (Laptop). Par défaut, NVDA est configuré pour un ordinateur de bureau. Pour passer en configuration ordinateur portable, allez dans le menu Préférences > Paramètres du clavier.

La configuration pour ordinateur de bureau fait un usage intensif du pavé numérique (avec le verrouillage numérique désactivé). Bien que la plupart des ordinateurs portables ne comportent pas de pavé numérique, certains peuvent l'émuler en maintenant enfoncée la touche FN puis en appuyant sur les lettres et les chiffres à la droite du clavier ( 7 8 9 u i o j k l etc). Si votre ordinateur ne vous permet pas de le faire, ou ne vous permet pas de passer en verrouillage numérique désactivé, vous pouvez alors utiliser la configuration pour ordinateur portable.

4.3. Gestes tactiles de NVDA

Si vous démarrez NVDA avec un périphérique contenant un écran tactile, et que vous utilisez Windows 8 ou supérieur, vous pouvez aussi contrôler NVDA dirrectement via l'écran tactile. Tant que NVDA est démarré, toutes les manipulations sur l'écran tactile vont dirrectement à NVDA. De plus, toutes les actions qui peuvent être faites normalement sans NVDA ne fonctionneront pas.

4.3.1. Explorer l'écran

L'action la plus basique que vous pouvez effectuer avec l'écran tactile est d'annoncer le texte ou l'élément à n'importequel point de l'écran. Pour se faire, placez un doigt n'importe où sur l'écran. Vous pouvez également garder votre doigt sur l'écran et le bouger autour pour lire les autres éléments et textes sur lesquels votre doigt se place.

4.3.2. Gestes tactiles

Lorsque les commandes NVDA seront décrites dans ce guide utilisateur, elles peuvent lister un gest tactile qui peut être utiliser pour activer cette commande avec l'écran tactile. Voici quelques instructions sur la manière d'effectuer les gestes tactiles requis pour effectuer les commandes NVDA avec un écran tactile.

Clic

Tapez rapidement sur l'écran avec un ou plusieurs doigts.

Taper avec un doigt est un simple clic. Cliquer avec deux doigts en même temps est un clic 2-doigts, et ainsi de suite.

Si le même clic est fait une fois ou plus encore dans une succession rapide, NVDA le prendra comme un geste tactile multi-clic. Cliquer deux fois sera un double-clic. Cliquer trois fois sera un triple-clic etc. Ces gestes reconnaissent également combien de doigts ont été utilisés, il est donc possible d'avoir des gestes tel qu'un triple-clic à deux doigts, ou un clic à quatre doigts, etc.

Glisser

Glicez rapidement votre doigt sur l'écran.

On peut glisser dans les quatre directions : glissé gauche, glissé droite, glissé bas, glissé haut.

Il est aussi possible d'utiliser plusieurs doigts. Il est donc possible d'avoir un geste à deux doigts vers le haut ou trois vers la droite.

4.3.3. Modes tactiles

Comme il y a plus de commandes NVDA que de gestes tactiles possibles, NVDA a plusieur modes tactiles entre lesquels on peut basculer, qui rendent un sertain nombre de commandes disponibles. Les modes actuellement existants sont le mode texte et le mode objet. Certaines commandes listées dans ce manuel peuvent avoir un mode listé entre parenthèses après le geste. Par exemple, glissé haut (mode texte) signifi que cette commande sera exécutée si vous glissez vers le haut, mais seulement en mode texte. Si la commande n'a aucun mode associé, il fonctionnera dans tous.

Pour basculer entre les modes, faites un clic avec trois doigts.

4.4. Aide des commandes

Des commandes claviers et tactiles sont mentionnées tout au long de ce guide utilisateur, , mais une manière simple d'explorer les différentes touches de commande est d'activer l'aide clavier.

Pour activer l'aide clavier, appuyez sur NVDA+1. Pour la désactiver, appuyez à nouveau sur NVDA+1. Quand l'aide clavier est activée, l'appui sur une touche Braille ou clavier, ou faire un geste à l'écran tactile annonce sa fonction (si elle en a une). La fonction n'est pas réellement exécutée, vous pouvez donc appuyer sur n'importe quelle touche sans crainte.

4.5. Le menu NVDA

Le menu NVDA vous permet de gérer les paramètres de NVDA, d'obtenir de l'aide, sauvegarder ou recharger la configuration, modifier le dictionnaire de prononciation, accéder à des outils additionnels ou quitter NVDA.

Pour accéder au menu NVDA depuis n'importe où dans Windows pendant que NVDA est en fonction, appuyez sur NVDA+n sur votre clavier, ou faites un double clic à deux doigts sur un écran tactile. Vous pouvez également y accéder par la barre des tâches système. Faites un clic droit sur l'icône de NVDA située dans la barre des tâches système, ou accédez à la barre des tâches système en appuyant sur Windows+b,,Appuyez sur flècheBas jusqu'à atteindre l'icône NVDA puis appuyez sur la touche Application généralement située à côté de la touche Control de droite. Quand le menu apparaît, vous pouvez utiliser les flèches pour naviguer dans le menu, et la touche entrée pour activer un élément.

5. Naviguer avec NVDA

NVDA vous permet de naviguer dans le système ou de l'explorer de différentes manières, soit en interaction normale, soit en revue.

5.1. Objets

Chaque application et le système d'exploitation lui-même se compose d'un grand nombre d'objets. Un objet est un élément simple tel qu'un morceau de texte, un bouton, une case à cocher, un curseur, une liste ou une zone d'édition.

5.2. Naviguer avec le Focus système

Le focus système, ou plus simplement focus, est l'objet qui reçoit les touches tapées au clavier. Par exemple, si vous tapez du texte dans une zone d'édition, cette zone d'édition a le focus.

La manière la plus commune de naviguer dans Windows avec NVDA, est de simplement se déplacer avec des commandes clavier normales, comme Tabulation et Majuscule+Tabulation pour se déplacer entre les contrôles, Alt pour aller au menu puis Flèches pour naviguer dans les menus, utiliser Alt+Tabulation pour passer d'une application à l'autre. Cela déplace le focus système, c'est-à-dire l'objet qui reçoit les touches que vous tapez au clavier. Quand vous faites cela, NVDA donne des informations sur l'objet ayant le focus, telles que son nom, son type, son état, sa valeur, sa description, son raccourci clavier et sa position.

Quelques touches de commande utiles pour naviguer avec le focus :

5.4. Naviguer par Objet

La plupart du temps, vous travaillerez avec les applications en utilisant des commandes qui déplacent le focus et le curseur. Cepandant, vous pouvez parfois avoir besoin d'explorer l'application en cours ou le système sans déplacer le focus. Vous pouvez aussi avoir besoin d'accéder à des objets inaccessibles par les commandes clavier habituelles. Dans ces cas, vous pouvez utiliser la navigation par objet.

La navigation par objet vous permet de vous déplacer parmi les différents (objets #Objects) et d'obtenir des informations à leur sujet. Quand vous atteignez un objet, NVDA l'annoncera de la même manière qu'il annonce le focus système. Si vous désirez explorer tout le texte tel qu'il se présente à l'écran, vous pouvez également utiliser la revue de l'écran.

Pour ne pas avoir à errer entre tous les objets présents dans le système, ceux-ci sont groupés hiérarchiquement. Cela signifie que vous devez entrer dans certains objets pour voir les objets qu'ils contiennent. Par exemple, une liste contient des éléments de liste, vous devez donc entrer dans la liste pour voir ces éléments. Si vous êtes sur un élément de liste, aller à l'élément suivant ou précédent vous amènera à un autre élément de la même liste. Si vous allez sur l'objet contenant les éléments de la liste, vous serez ramené à la liste. Vous pourrez alors la quitter si vous désirez accéder à d'autres objets. De la même manière, une barre d'outils contient des contrôles, vous devrez donc entrer dans la barre d'outils pour accéder aux contrôles qu'elle contient.

L'objet en cours de revue s'appelle l'objet navigateur. Quand vous atteignez un objet, vous pouvez examiner son contenu en utilisant les commandes de revue de texte en étant en mode Revue d'objet. Par défaut, l'objet navigateur suit le focus système, mais ce comportement peut être désactivé.

En étant en mode revue d'objet, vous pouvez revoir seulement le contenu de l'objet du navigateur. Pour les objets comme les champs d'édition ou les documents basiques, cela sera généralement le texte conteni. Pour les autres objets, cela eut être le nom ou la valeur.

5.6.2. Revue de document

Quand l'objet du navigateur est dans un document comme une page web ou un document complex (ex : document Lotus Symphony), il est possible de passer en mode revue de document. Le mode revue de document vous permet de relire le contenu du document en entier.

En passant du mode revue d'objet au mode revue de document, le curseur de revue se place là où se trouve le navigateur d'objet. En bougeant le curseur de revue, le navigateur d'objet est déplacé à la même position que le curseur de revue.

Notez que NVDA basculera automatiquement au mode revue de document depuis le mode revue objet en bougeant dans le document en mode navigation.

5.6.3. Revue de l'écran

Le mode revue de l'écran vous permet de lire le texte visible tel qu'il aparaît à l'écran dans l'application courrante. Cela est très cimilaire au fonctionnalités mode revue ou curseur souris des autres lecteur d'écran sous Windows.

En basculant en mode revue de l'écran, le curseur de revue est placé à la position de l'objet du navigateur. En bougeant sur l'écran avec le curseur de revue, le navigateur d'objet est déplacé à l'objet le plus proche.

Notez que dans certaines nouvelles applications, NVDA ne pourra pas lire tout ou une partie du texte à l'écran, étant donné qu'elles utilise des techniques d'affichage qui ne peuvent être gérées actuellement.

8. Configurer NVDA

8.1. Préférences

La plupart des paramètres de NVDA peuvent être modifiés en utilisant les boîte de dialogue accessibles par le sous-menu Préférences du menu NVDA. Dans toutes les boîtes de dialogue de NVDA, appuyez sur le bouton Accepter pour valider vos modifications. Pour annuler les modifications, appuyez sur le bouton Annuler ou sur la touche échap. Certains paramètres peuvent aussi être modifiés en utilisant des raccourcis clavier que vous trouverez dans les sections ci-dessous.

8.1.1. Paramètres Généraux (NVDA+control+g)

La boîte de dialogue Paramètres généraux se trouve dans le menu Préférences. Elle contient les options suivantes :

Langue

C'est une liste déroulante qui permet de choisir la langue de l'interface utilisateur et des messages de NVDA. La liste comporte de nombreuses langues et le dernier élément est dénommé "Windows". Ce choix indique à NVDA d'utiliser la langue de Windows.

Noter qu'il faut relancer NVDA quand on change la langue. NVDA vous demandera si vous désirez le redémarrer quand vous changez la sélection. Appuyez sur Accepter et NVDA redémarrera.

Sauvegarder la configuration en quittant

C'est une case à cocher qui, une fois cochée, indique à NVDA de sauvegarder automatiquement la configuration courante lorsqu'on arrête NVDA.

Avertir avant de quitter NVDA

Cette option est une case à cocher vous permettant d'indiquer si une confirmation doit être demandée quand vous voulez quitter NVDA. Quand elle est cochée, un dialogue apparaîtra quand vous voudrez quitter NVDA vous demandant si oui ou non vous voulez quitter.

Niveau de journalisation

C'est une liste déroulante qui vous permet de choisir la quantité d'information que NVDA consignera dans son journal durant son exécution. Généralement, l'utilisateur n'aura pas besoin de modifier ce paramètre dans la mesure où peu d'informations sont consignées. Cependant, si vous voulez fournir des informations lors d'un rapport d'erreur, il est peut-être bon d'augmenter cette valeur.

Démarrer NVDA automatiquement après ma connexion à Windows

Si cette option est activée, NVDA démarrera automatiquement dès que vous vous connecterez à Windows. Cette option n'existe que dans les versions installées de NVDA.

Utiliser NVDA sur l'écran de connexion à Windows (nécessite des privilèges administrateur)

Si vous vous connectez à Windows en fournissant un nom d'utilisateur et un mot de passe, l'activation de cette option permettra à NVDA de démarrer automatiquement dès l'écran de connexion au démarrage de Windows. Cette option n'existe que dans les versions installées de NVDA.

Utiliser les paramètres actuellement sauvegardées sur l'écran de connexion et les autres écrans sécurisés

L'appui sur ce bouton copie votre configuration utilisateur actuellement sauvegardée dans le répertoire système de NVDA, ainsi, NVDA peut l'utiliser quand il s'exécute sur l'écran de connexion et les autres écrans sécurisés. Pour être sûr que tous vos paramètres sont transférés, sauvegardez d'abord vos paramètres avec la commande NVDA+control+c ou par Sauvegarder la configuration dans le menu NVDA. Cette option n'existe que dans les versions installées de NVDA.

8.1.3. Paramètres vocaux (NVDA+control+v)

Le dialogue Paramètres vocaux qui se trouve dans le menu Préférences, contient des options vous permettant de changer le son de la voix. Pour une manière plus rapide de contrôler les paramètres vocaux depuis n'importe où, reportez-vous à la section boucle des paramètres synthétiseur.

Ce dialogue contient les options suivantes :

Voix

Une liste déroulante qui donne la liste de toutes les voix du synthétiseur courant. A l'aide des flèches de direction, on peut entendre les différents choix. Les flèches gauche et haute permettent de reculer d'un élément, droite et bas d'avancer.

Variante

Une liste déroulante permettant de choisir la variante utilisée par le synthétiseur. Actuellement, seule eSpeak propose ce paramètre. Les variantes eSpeak sont comparables à des voix, dans la mesure où elles modifient significativement les attributs de la voix eSpeak. Les variantes sont par exemple homme ou femme.

Débit

Cette option vous permet de modifier le débit de la parole. C'est un potentiomètre gradué de 0 à 100, (0 étant la vitesse la plus lente et 100 la plus rapide).

Hauteur

Cette option vous permet de modifier la hauteur de la voix. C'est un potentiomètre gradué de 0 à 100, (0 pour le son le plus grave et 100 pour le plus aiguë).

Volume

Cette option est un potentiomètre gradué de 0 à 100 qui permet de modifier le volume de la voix, (0 étant le volume le plus bas et 100 le plus élevé).

Inflexion

Un potentiomètre vous permettant de choisir le niveau d'inflexion (variation en hauteur de la voix) que le synthétiseur devra utiliser.

Changement automatique de langue

Cette case à cocher vous permet de choisir si NVDA doit changer de langue à la volée si une balise de langue est présente dans le texte en cours de lecture. Par défaut, cette option est activée. À l'heure actuelle, seul le synthétiseur eSpeak prend en charge le changement automatique de langue.

Changement automatique de Dialecte

Si le changement automatique de langue est activé, cette case à cocher vous permet de choisir si les changements de dialecte doivent être pris en compte en plus des changements de langues réels. Par exemple, un texte peut être en Anglais Américain et comporter des passages marqués comme Anglais Britannique. Si cette option est activée, le synthétiseur respectera les changements d'accent. Par défaut, cette option est désactivée.

Niveau des symboles et ponctuations

Key: NVDA+p

Ceci vous permet de choisir la quantités de ponctuations et autres symboles qui seront prononcés. Par exemple, si vous choisissez Tous, tous les symboles seront annoncés. Cette option s'applique à tous les synthétiseurs, pas uniquement au synthétiseur courant.

Pourcentage de changement de la hauteur pour indiquer les majuscules

Ce champ d'édition vous permet d'entrer la valeur du changement de hauteur de la voix à l'annonce d'une lettre majuscule. Cette valeur est un pourcentage, une valeur négative réduit la hauteur, une valeur positive l'augmente. Pour n'avoir aucun changement de hauteur, entrez la valeur 0.

Dire "maj" avant les majuscules

Une case à cocher qui, cochée, indique à NVDA de dire maj pour une lettre en majuscule quand on l'atteint par les flèches de direction ou lors de la frappe. Normalement, NVDA énonce chaque lettre majuscule avec un ton plus aiguë, mais ceci n'est pas correctement pris en charge par certains synthétiseurs, si bien que cette option est parfois utile.

Annoncer les majuscules par des bips

Quand cette case à cocher est cochée, NVDA émettra un petit bip chaque fois qu'il rencontre un caractère majuscule. Comme l'option "Dire maj avant les majuscules", cette option est utile pour les synthétiseurs ne pouvant pas modifier la hauteur de leur voix pour les majuscules.

Utilisez la fonction d'épellation si elle est supportée

Certains mots ne comportent qu'un seul caractère, mais la prononciation est différente selon que le caractère est prononcé comme un caractèreunique (épellation) ou comme un mot. Par exemple, en Français, "y" est à la fois une lettre et un mot. Cette option permet au synthétiseur de différencier les deux cas si le synthétiseur le supporte. La plupart des synthétiseurs le supportent.

En général, cette option devrait être activée. Cependant, certains synthétiseurs utilisant les API de parole Microsoft se comportent bizarrement quand cette option est activée. Si vous rencontrez des problèmes à l'épellation de caractères, essayez de désactiver cette option.

Annoncer les raccourcis clavier des objets

Quand cette case est cochée, NVDA inclut, à l'annonce d'un objet ou d'une commande, le raccourci clavier qui lui est associé. Par exemple, le menu fichier d'une barre de menu peut avoir pour raccourci clavier alt+f.

Deviner le rang de l'objet quand cette information n'est pas disponible

Si l'annonce du rang de l'objet dans une liste est activée, cette option autorise NVDA à deviner le rang de l'objet si cette information n'est pas disponible pour un objet particulier.

Quand l'option est activée, NVDA annonce le rang pour un plus grand nombre de contrôles tels que les menus et les barres d'outils. Cependant, cette information peut parfois être inexacte.

8.1.10. Paramètres de Saisie

La boîte de dialogue Paramètres de Saisie se trouve dans le menu préférences. Cette boîte de dialogue permet de contrôler comment NVDA rapporte la saisie des caractères asiatiques, avec IME ou les services de texte et langues d'entrée. Notez bien que comme les méthodes de saisie varient beaucoup dans les possibilitées disponibles et leur rendu d'informations, il peut être utile de refaire cette configuration pour chaque méthode d'entrée afin de bénéficier de la meilleure utilisation.

Annonce automatique de toutes les suggestions disponibles

Cette option, activée par défaut, permet de choisir si oui ou non toutes les suggestions visibles doivent être automatiquement rapportés quand une liste de suggestions aparaît ou quand sa page change. Activer cette option sur les méthodes d'entrée pictographic tels que chinois Nouveau ChangJie ou Boshiami est utile car vous pouvez entendre automatiquement tous les symboles et leurs numéros, et vous pouvez en choisir un tout de suite. Toutefois, pour les méthodes de saisie phonétique comme le nouveau chinois phonétique, il peut être plus utile de désactiver cette option puisque tous les symboles ont les mêmes sons et vous devrez utiliser les touches fléchées pour naviguer dans les éléments de la liste individuellement pour obtenir plus d'informations de la descriptions de caractères pour chaque suggestion.

Annoncer la suggestion sélectionnée

Cette option, activée par défaut, vous permet de choisir si NVDA doit ou non annoncer la suggestion sélectionnée lorsqu'une liste de suggestion apparaît ou quand la sélection est changée. Pour les méthodes ou la suggestion peut être changée en utilisant les flèches (comme chinois nouveau phonétique) cela est nécessaire, cependant pour sertaines méthodes il est préférable de désactiver cette option. Notez que même si cette option est désactivée, le curseur de revue se placera toujours sur la suggestion sélectionnée, vous permettant toujours d'utiliser la navigation par objet ou le curseur de revue pour sélectionner la suggestion.

Toujours inclure une description courte du caractère pour les suggestions

Cette option, activée par défaut, permet de choisir si oui ou non NVDA doit fournir une description courte de chaque caractère d'une suggestion, quand elle est sélectionnée ou lors de la lecture de la liste quand elle apparaît. Notez que pour les langues comme le chinois, l'annonce des description de caractères suplémentaires pour la suggestion sélectionnée n'est pas affectée par cette option. Cette option est utile pour les méthodes d'entrées Coréennes et Japonnaises.

Annonce les changements dans la chaîne en cours de lecture

Certains modes de saisie telles que le chinois ChangJie phonétique et Nouveau ChangJie peuvent avoir une chaîne de lecture (parfois connu sous le nom d'une chaîne précomposition). Avec cette option, vous pouvez choisir si oui ou non NVDA doit annoncer les nouveaux caractères saisis dans la chaîne de lecture en cours. Cette option est activée par défaut. Notez que certaines ensiènes méthodes comme le Chinois ChangJie n'utilisent pas la chaîne en cour de lecture pour les caractères de précomposition, utilisez directement la chaîne de composition. Voyez l'option suivante pour configurer l'annonce de la chaîne de composition.

Annoncer les changements dans la chaîne de composition

Après que les données de lecture ou de précomposition aient été combinées en un symbole pictographique valide, la plupart des méthodes d'entrée placent ce symbole dans une chaîne de composition pour stoquer temporairement avec les autres symboles combinés avant de les insérer dans le document. Cette option permet de choisir si oui ou non NVDA doit annoncer tous les symboles quand ils apparaissent dans la chaîne de composition. Cette option est activée par défaut.

8.1.13. Dictionnaires de prononciation

Le menu "dictionnaires de prononciation" du menu "préférences" contient des dialogues qui vous permettent de gérer la manière dont NVDA prononce certains mots ou phrases. Il y a actuellement trois types de dictionnaires. Ce sont :

  • Défaut : les règles de ce dictionnaire affectent la parole dans tout NVDA.
  • Voix : les règles de ce dictionnaire n'affectent que la voix en cours d'utilisation.
  • Temporaire : les règles de ce dictionnaire affectent tout NVDA mais seulement pour la session en cours.

Tous les dialogues des dictionnaires contiennent une liste de règles qui seront utilisées pour produire la parole. Chaque dialogue contient aussi des boutons "Ajouter", "Editer" et "Supprimer"

Pour ajouter une règle au dictionnaire, cliquez sur le bouton "Ajout", remplissez les champs dans la boîte de dialogue puis cliquez sur "Accepter". Vous verrez ainsi votre nouvelle règle dans la liste des règles. Cependant, pour être sûr que votre règle est vraiment sauvegardée, n'oubliez pas de cliquer sur le bouton "Accepter" pour quitter le dialogue de dictionnaire une fois que vous aurez fait toutes vos modifications.

Les règles des dictionnaires de NVDA vous permettent de remplacer une chaîne de caractères par une autre. Par exemple, vous pourriez vouloir que NVDA prononce "le mot grenouille" chaque fois qu'il rencontre le mot "oiseau". Dans le dialogue d'ajout de règles, la manière la plus simple de faire cela est d'entrer le mot "oiseau" dans la zone "modèle" et le mot "grenouille" dans la zone de remplacement. Vous pouvez aussi ajouter un commentaire décrivant la règle dans la zone commentaire, quelque chose comme "Remplacer oiseau par grenouille".

Cependant, les dictionnaires de NVDA font bien mieux que le simple remplacement de mot. Le dialogue Ajout de règles contient une case à cocher permettant de choisir si vous voulez que la règle respecte la casse pour que NVDA tienne compte des majuscules et minuscules. Par défaut, NVDA ne tient pas compte de la casse. Une autre case à cocher vous permet de définir si votre modèle est une expression régulière. Une expression régulière est un modèle contenant des symboles particuliers vous permettant de trouver une correspondance sur plus d'un caractère à la fois, ou seulement sur les lettres ou les chiffres etc. Les expressions régulières ne sont pas expliquées dans ce manuel, mais on trouve, sur le web, beaucoup de tutoriels qui vous donneront plus d'informations.

8.1.14. Prononciation des symboles et ponctuations

Ce dialogue vous permet de modifier la façon dont les ponctuations et autres symboles sont prononcés ainsi que le niveau de symbole auquel ils sont annoncés.

Pour modifier un symbole, sélectionnez-le d'abord dans la liste de symboles. Le champ Remplacement vous permet de modifier le texte qui doit être prononcé à la place de ce symbole. En utilisant le champ Niveau, vous pouvez ajuster le niveau minimum auquel ce symbole doit être annoncé.

Quand vous avez fini, cliquez sur le bouton Accepter pour sauvegarder vos modifications, ou le bouton Annuler pour les supprimer.

8.3. Fichiers de configuration et profils utilisateur

Les versions portables de NVDA enregistrent tous les paramètres, modules d'applications personnalisés et pilotes personnalisés dans un répertoire nommé userConfig situé dans le répertoire de votre copie portable de NVDA.

Les versions installées de NVDA enregistrent tous les paramètres, modules d'applications personnalisés et pilotes personnalisés dans un répertoire spécial de NVDA situé dans votre profil utilisateur Windows. Cela signifie que chaque utilisateur du système peut avoir ses propres paramètres NVDA. Pour accéder à votre répertoire de paramètres pour une version installée de NVDA, depuis le menu Démarrer vous pouvez aller à Tous les programmes -> NVDA -> Explorer les paramètres utilisateur.

Les paramètres qu'utilise NVDA quand il s'exécute sur l'écran de connexion ou l'écran UAC sont enregistrés dans le répertoire systemConfig situé dans le répertoire nVDA. En général, cette configuration n'a pas à être modifiée. Pour modifier cette configuration, configurez nVDA à votre convenance puis demandez à NVDA de copier la configuration sur l'écran de connexion en utilisant le bouton à cet effet dans le dialogue Paramètres généraux.

9. Outils Additionnels

9.1. Visionneuse du journal

La visionneuse du journal, située dans Outils du menu NVDA, vous permet de consulter tous les événements enregistrés depuis votre dernier démarrage de NVDA.

En plus de lire le contenu, vous pouvez également sauvegarder une copie du journal ou réactualiser la visionneuse de manière à voir les événements les plus récents. Ces actions sont disponibles dans le menu de la visionneuse.

9.2. Visionneuse de Parole

Pour les développeurs voyants ou les personnes présentant NVDA à un public voyant, une fenêtre flottante est disponible, affichant tout ce que dit NVDA.

Pour activer la visionneuse de parole, cochez l'élément de menu Visionneuse de parole situé dans Outils dans le menu NVDA. Décochez l'élément de menu pour la désactiver.

Pendant que la visionneuse de parole est active, elle se met constamment à jour vous permettant de voir les dernières paroles prononcées par NVDA. Cependant, si vous cliquez ou mettez le focus sur la visionneuse, NVDA arrêtera momentanément de mettre à jour le texte, vous pourrez ainsi sélectionner ou copier le contenu.

9.3. Gestionnaire de Modules Complémentaires

Le Gestionnaire de Modules Complémentaires, accessible en sélectionnant Gestion des Modules Complémentaires dans le menu Outils du menu NVDA, permet d'installer ou de désinstaller des paquets de Modules Complémentaires pour NVDA. Ces paquets sont fournis par la communauté et contiennent du code qui peut ajouter ou modifier des fonctionnalités dans NVDA ou proposer le support de plages brailles ou synthèses vocales suplémentaires.

Le gestionnaire de Modules Complémentaires contient une liste affichant tous les Modules Complémentaires actuellement installés dans votre configuration utilisateur de NVDA. Un nom de paquet, la version et l'auteur sont affichés pour chaque Module Complémentaire, et des informations suplémentaires telle qu'une description et une URL peuvent être affichées en sélectionnant le Module Complémentaire et en activant le bouton À propos du Module Complémentaire...

Pour télécharger des modules complémentaires mis à disposition par la comunauté, cliquez sur le bouton Obtenir des modules complémentaires. Ce bouton ouvre la page des modules complémentaires NVDA. Si NVDA est installé sur votre système, vous pourrez ouvrire le module complémentaire directement depuis votre navigateur à fin de démarrer le processuce d'installation décrit plus loin. Autrement, enregistrez le fichier du module complémentaire et suivez les instructions ci-dessous.

Pour ajouter un Module Complémentaire, cliquez sur le bouton Ajouter. Vous pourrez ainsi rechercher le paquet du Module Complémentaire (fichier .nvda-addon) sur votre ordinateur ou votre réseau. Après avoir cliquer sur Ouvrir, le processuce d'installation commence.

Quand un module complémentaire est sur le point d'être installé, NVDA vous demandera de confirmer que vous souhaitez installer ce module complémentaire. Comme les fonctionalités des Modules Complémentaires ne sont pas restraintes dans NVDA, ce qui implique en théorie qu'ils peuvent accéder à vos données ou à tout votre système pour une version installée, il est important de n'installer que des Module Complémentaires venant d'une source fiable. Dès que le Module Complémentaire est installé, NVDA doit redémarrer pour que le Module Complémentaire puisse s'activer. Ce Module Complémentaire aura l'état installer dans la liste jusqu'à ce que vous relanciez NVDA.

Pour supprimer un Module Complémentaire, sélectionnez-le dans la liste et cliquez sur le bouton Supprimer. NVDA vous demandera de confirmer la désinstallation. Comme pour l'installation, NVDA doit être relancé pour que le Module Complémentaire soit complettement supprimé. Son statu dans la liste sera supprimer.

Le gestionnaire a également un bouton Fermer pour fermer la boîte de dialogue. Si vous avez installé ou supprimé des Modules Complémentaires, NVDA vous demandera d'abord si vous voulez le redémarrer pour que vos changements prennent effet.

Jusqu'à maintenant, il était possible d'étendre les fonctionnalités de NVDA en copiant des plugins ou pilotes dans votre répertoire de configuration utilisateur. Même si cette version peut toujours les charger, ceux-ci ne seront pas affichés dans le gestionnaire de Modules Complémentaires. Il est recommendé de supprimer ces fichiers de votre répertoire de configuration et d'installer le Module Complémentaire approprié s'il y en a un.

Le synthétiseur eSpeak est intégré dans NVDA et ne dépend d'aucun pilote ou programme additionnel pour être installé. Par défaut, NVDA démarre en utilisant eSpeak. Chaque voix fournie avec eSpeak correspond à une langue différente. Espeak prend en charge plus de 20 langues.

Vous disposez également d'un grand choix de variantes, permettant de modifier considérablement le son de la voix.

10.2. Microsoft Speech API version 4 (SAPI 4)

SAPI 4 est un ancien standard Microsoft pour les synthétiseurs de parole logiciels. Beaucoup de synthétiseurs de parole compatibles avec ce standard peuvent être achetés ou téléchargés gratuitement auprès de différentes sociétés ou sites web. Quand vous utilisez ce synthétiseur avec NVDA, la liste des voix disponibles (accessibles via le dialogue Paramètres vocaux, ou la boucle des paramètres synthétiseur) contient toutes les voix de tous les synthétiseurs SAPI 4 installés sur votre système.

Si vous avez des voix SAPI 4 installées mais que le synthétiseur n'apparaît pas dans la liste des synthétiseurs de NVDA, veuillez installer SAPI 4.0 runtime binaries, disponible sur http://activex.microsoft.com/activex/controls/sapi/spchapi.exe.

10.3. Microsoft Speech API version 5 (SAPI 5)

SAPI 5 est un standard Microsoft pour les synthétiseurs de parole logiciels. Beaucoup de synthétiseurs de parole compatibles avec ce standard peuvent être achetés ou téléchargés gratuitement auprès de différentes sociétés ou sites web, mais il est probable que votre système possède déjà une voix SAPI 5 préinstallée. Quand vous utilisez ce synthétiseur avec NVDA, la liste des voix disponibles (accessibles via le dialogue Paramètres vocaux, ou la boucle des paramètres synthétiseur) contient toutes les voix de tous les synthétiseurs SAPI 5 installés sur votre système.

La plateforme Microsoft Speech fournit des voix pour beaucoup de langues, normalement utilisées dans le développement de serveurs vocaux. Ces voix peuvent également être utilisées avec NVDA.

12. Fonctions Avancées

12.1. Redéfinir les Assignations de Touches et les Autres Gestes de Commande

L'utilisateur peut ajouter ou réassigner des gestes de commande (telles que des touches de commande) aux scripts dans un fichier spécial situé dans le répertoire de configuration NVDA de l'utilisateur. Ce fichier s'appelle gestures.ini.

Ce fichier utilise la syntaxe standard des fichiers .ini. Le fichier peut contenir plusieurs sections et chaque section peut contenir une ou plusieurs entrées.

Chaque section contient les assignations pour un script dans un module et une classe python particulier de NVDA.

  • Le nom de la section doit être le nom du module et la classe séparés par un point (.).
  • La clé de chaque entrée est le nom du script auquel le geste de commande doit être assigné. Alternativement, on peut utiliser None pour désassigner un geste d'un script auquel il était précédemment assigné. Chaque entrée ne peut apparaître qu'une fois par section, y compris None.
  • La valeur de chaque entrée est une liste d'identifiants de gestes séparés par une virgule, pour chaque geste de commande qui doit être assigné. Les identifiants de geste terminés par une virgule doivent être entourés de quotes (" ou ').

Les identifiants de geste comprennent un code de matériel sur deux lettres (kb pour clavier, br pour terminal braille et ts pour écran tactile), un sous-code une disposition entre parenthèses, le signe deux-points, puis un ou plusieurs noms de touches Braille, clavier ou tactile séparés par le signe plus (+).

  • Pour un raccourci clavier, le code est kb. La partie entre parenthèses correspond à la disposition clavier et est optionnelle. Si elle n'est pas indiquée, le geste s'appliquera à toutes les configurations clavier. La chaîne après la virgule est un ou plusieurs noms de touches du clavier séparées par un signe plus (+).
  • Pour les gestes s'appliquant au terrminal braille, le code est br. La partie entre parenthèses indique le terminal braille spécifique est est obligatoire. La chaîne après la virgule est un ou plusieurs noms de touches du terminal Braille séparées par un signe plus (+).
  • Pour les gestes tactiles, le code est ts. La partie entre parenthèses spécifie le mode tactile à utiliser et est optionnelle. La partie après la virgule est un geste tactile comme double_tap (double clic), 2fingerflickUp (Glisser vers le haut 2-doigts), ou 3finger_tap (clic 3-doigt) par exemples.

Pour découvrir les identifiants de gestes, les noms de scripts, les noms de classes et de modules qui les contiennent, vous pouvez :

  1. Activer l'aide des commandes.
  2. Activer le geste (appuyez sur la touche, toucher l'écran tactile etc).
  3. Désactiver l'aide des commandes.
  4. Activer la visionneuse de journal dans le menu outils de NVDA.
  5. Examiner les entrées récentes du journal. L'une d'entre elle devrait vous donner des informations au sujet du geste de commande que vous venez d'entrer, incluant le module, la classe et le script s'il est assigné.

Voici un exemple de la manière dont vous pourriez assigner NVDA+maj+t au script donnant la date et l'heure.

Pour trouver le nom du script, module.class pour date time, vous devez activer l'aide des commandes puis appuyer sur NVDA+f12 qui est le geste standard pour le script date time. Désactivez ensuite l'aide des commandes puis ouvrez la visionneuse de journal.

Vers la fin du journal, vous devriez voir : 

  INFO - inputCore.InputManager._handleInputHelp (13:17:22):
  Input help: gesture kb(desktop):nvda+f12, bound to script dateTime on globalCommands.GlobalCommands

Vous pouvez en déduire que le nom du script est dateTime et que le module.class est globalCommands.globalCommands.

Si le fichier n'existe pas encore, vous devez créer un fichier nommé gestures.ini dans le répertoire de configuration utilisateur et y insérer les lignes suivantes : 

  [globalCommands.GlobalCommands]
  	dateTime = kb:NVDA+maj+t

Cela assignera la combinaison NVDA+maj+t (dans toutes les configurations de clavier) au script dateTime.

Notez que l'assignation originale NVDA+f12 fonctionnera toujours. Si vous voulez supprimer cette assignation, ajoutez la ligne suivante : 

  	None = kb:NVDA+f12

Bien que vous puissiez assigner un script à n'importe quelle touche, il peut être problématique d'utiliser la touche alt du clavier. NVDA passe les touches telles que alt, maj et ctrl au système d'exploitation même si elles doivent déclencher un script. Ainsi, si vous utilisez la touche alt dans une combinaison, il se peut qu'elle ouvre la barre de menu tout en déclenchant le script. il est donc préférable de n'utiliser que maj, ctrl ou la touche NVDA.

12.2. Personnalisation Avancée de la Prononciation des Symboles

Il est possible de personnaliser la prononciation des ponctuations et autres symboles au-delà de ce que permet le dialogue Prononciation des symboles et ponctuations. Par exemple, vous pouvez indiquer si un symbole particulier doit être envoyé au synthétiseur pour créer une pause ou modifier une inflexion et vous pouvez ajouter vos propres symboles.

Pour faire cela, vous devez éditer le fichier contenant les informations de prononciation des symboles dans votre répertoire utilisateur NVDA. Le fichier s'appelle symbols-xx.dic où xx est le code de la langue. Le format de ce fichier est documenté dans la section Prononciation des Symboles du guide de développement NVDA disponible dans la section Développement du site NVDA. Cependant, l'utilisateur ne peut pas définir de symboles complexes.
